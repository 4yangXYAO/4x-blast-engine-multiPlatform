## **Nada (Tone) Guidelines**

1. **Formal** (untuk error, validasi): Gunakan "Anda", "Silakan", "Harap"
   - "Silakan periksa kredensial Anda."
   
2. **Friendly** (untuk success, info): Gunakan "Terima kasih", "Selamat", etc
   - "Halo! Kampanye Anda berhasil dikirim!"

3. **Clear & Actionable**:
   - Always include "what to do next"
   - Never say just "Error occurred"
   - Example: "Gagalâ€”cek log untuk detail" vs "Gagal. Cek log di bagian atas halaman untuk melihat detail errornya, lalu perbaiki kredensial."

## **Catatan Umum**

- **Jangan ever use**: "You" (Anda formal OK, tapi not excessive), slang like "goblok", "wtf", etc. This is professional tool.
- **Keep technical identifiers**: variable names, function names, class names, endpoint paths, API names (GraphQL, REST, Express, JWT, etc)
- **Localize UI only**: User visible text (buttons, labels, titles, messages, tooltips, placeholders, error texts)
- **Do not translate**: File names, directory names, code strings used for internal identification
- **Use consistent spacing**: No double spaces after period, use standard Indonesian punctuation.
